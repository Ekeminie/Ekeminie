## 🏗️ Architecture Philosophy

- **Separation of concerns first** — UI never knows about business logic
- **BLoC/Cubit for predictability** — state changes are always traceable and testable
- **Feature-first folder structure** — scales with team size
- **CI/CD from day one** — not a "nice to have", a delivery multiplier
- **Testability baked in** — unit, widget, and integration tests at every layer
